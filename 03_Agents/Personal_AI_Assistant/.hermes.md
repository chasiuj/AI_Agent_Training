## ⚠️ 鐵則：見到危險指令立刻停

| 見到這些關鍵字 | 動作 |
|--------------|------|
| `delete` / `del` | 停 |
| `rm` / `rm -rf` | 停 |
| `force` / `--force` / `-f` | 停 |

**真要移除，先用以下安全方式：**
1. **Bin**（垃圾桶）— 可還原
2. **Archive** — 壓縮備份後再刪
3. **Rename** — 改名而非刪除
4. **Backup** — 確認備份後再執行

> **任何包含刪除意圖的指令，在執行前都必須先確認是否已用安全方式處理。**

## 三層保護框架

### 🗑️ 第一層：垃圾桶保護
**預設動作：刪除 → 垃圾桶（可還原）**

**目的：** AI 跟人一樣會打錯字。`rm -rf node_modules /` 多了一個空格，就由「刪一個 folder」變成「刪系統根目錄」。

**實作方式：**
- 將 `rm` / `del` / `delete` 替換為 `trash-put`（垃圾桶指令）
- 誤刪可以從垃圾桶還原
- **不論任何理由，見到 `rm` / `rm -rf` / `del` / `delete` / `force` / `--force` 就停**

**⚠️ 鐵則：**
> 見到 `delete`、`rm`、`rm -rf`、`force` → 立刻停用
> 真要移除：先用 Bin → Archive → Rename → Backup

### 🚫 第二層：危險指令黑名單
直接拒絕「一執行就回不來」的指令，連問都不會問。

**被封鎖的指令（完整清單）：**
| 指令 | 風險 |
|------|------|
| `git reset --hard` | 直接清空未存檔的改動 |
| `git push --force` / `git push -f` | 覆蓋遠端紀錄 |
| `rm -rf /` | 刪除系統根目錄 |
| `dd if=/dev/zero of=/dev/sda` | 硬碟覆寫 |
| `mkfs` / `mkfs.ext4` 等 | 格式化 |
| `shutdown` / `halt` / `poweroff` | 關機 |
| `chmod -R 000` | 移除所有權限 |

**實作方式：** Hermes 的 `tirith` 預執行掃描

**目前狀態：** ✅ 已啟用
```yaml
security:
  tirith_enabled: true
  tirith_fail_open: false
  tirith_path: C:/Users/hcsiu/AppData/Local/hermes/bin/tirith
```

**⚠️ 鐵則：**
> 見到黑名單指令 → 不問，直接拒絕執行

### 🔐 第三層：權限模式
逐次彈確認框太煩，全部關掉太危險。由自己選確認程度。

**三種模式：**

| 模式 | 行為 | 建議 |
|------|------|------|
| `manual` | 所有危險指令都問確認 | 預設值 |
| `smart` ⭐ | AI 自動判斷：低風險直接執行，高風險才問 | **推薦** |
| `off` | 繞過所有確認 | ❌ 不建議 |

**目前狀態：** ✅ 已設定為 `smart`
```yaml
approvals:
  mode: smart
```
